NexLayer Documentation

Production-ready Next.js 15 SaaS landing page template with dark/light mode, Framer Motion animations, TypeScript, and Tailwind CSS.

Introduction

NexLayer is a premium Next.js template designed for SaaS products, developer tools, and AI applications. It ships with three homepage variants, 28 section components, 13 UI primitives, a complete blog system, and a production-quality design token system.

TechVersionNotes
Next.js^15.0App Router, SSR, Static Generation
React^19.0
TypeScript^5.0Strict mode enabled
Tailwind CSS^3.4CSS variable–based design tokens
Framer Motion^11.0Scroll-triggered animations
Lucide React^0.400Icon library

Requirements

  • Node.js 18.17+ (LTS recommended)
  • npm 9+ / pnpm 8+ / yarn 4+
  • A modern browser (Chrome, Firefox, Safari, Edge)

Installation

1. Install dependencies

npm install
# or
pnpm install

2. Set up environment variables

cp .env.example .env.local

Open .env.local and fill in your values (see Environment Variables).

3. Start the dev server

npm run dev

Open http://localhost:3000 in your browser.

Brand & Colors

The primary brand color is green (#22c55e). To change it:

1. Update Tailwind config

Edit tailwind.config.ts, replacing the brand color scale:

// tailwind.config.ts
brand: {
  50:  '#eff6ff',  // replace with your lightest tint
  ...
  500: '#3b82f6',  // ← your primary brand color
  ...
}

2. Update icon.tsx and opengraph-image.tsx

Both files hardcode #22c55e. Update to match your brand:

// app/icon.tsx  &  app/opengraph-image.tsx
background: '#3b82f6',   // ← your brand color

Typography

Three font families are configured via next/font/google in app/layout.tsx:

VariableFontUsage
--font-displaySynefont-display — headings, hero, logo
--font-bodyDM Sansfont-body — body text (default)
--font-monoJetBrains Monofont-mono — code, terminal

To swap fonts, replace the import in app/layout.tsx with any Google Font and update the variable name.

Dark / Light Mode

Mode is controlled by adding/removing the .dark class on <html>. ThemeProvider (components/ThemeProvider.tsx) persists the user's preference in localStorage.

The full surface color scale (50–950) is defined as CSS variables in app/globals.css and automatically inverts between dark and light mode.

The template defaults to dark mode. To default to light mode, change className="dark" to className="light" in app/layout.tsx.

Forms & API Routes

Two API routes are included out of the box:

RouteMethodPayload
/api/contactPOST{ name, email, subject, message }
/api/newsletterPOST{ email }

Both routes validate input and return { success: true } on success or { error: "message" } with a 4xx status on failure.

By default the routes only validate and log in development. Add your email service in the commented blocks inside each route file.

Email Integration

The recommended provider is Resend. Install the SDK, then uncomment the example block in the route file:

npm install resend
// app/api/newsletter/route.ts
import { Resend } from 'resend'
const resend = new Resend(process.env.RESEND_API_KEY)
await resend.contacts.create({
  email,
  audienceId: process.env.RESEND_AUDIENCE_ID!,
})

Alternative providers: Mailchimp, ConvertKit, Loops.so, Nodemailer. See comments in each route file.

Authentication

The login page UI is ready. Connect your chosen provider in app/login/page.tsx:

// Clerk
import { useSignIn } from '@clerk/nextjs'

// NextAuth
import { signIn } from 'next-auth/react'
await signIn('credentials', { email, password })

// Supabase
import { createClientComponentClient } from '@supabase/auth-helpers-nextjs'
await supabase.auth.signInWithPassword({ email, password })

Deployment

Vercel (recommended)

npm i -g vercel
vercel

Set environment variables in the Vercel dashboard under Settings → Environment Variables.

Other platforms

npm run build
npm run start   # starts Node.js server on port 3000

NexLayer works on any platform that supports Node.js 18+: Railway, Render, Fly.io, AWS, DigitalOcean.

Always set NEXT_PUBLIC_APP_URL to your production domain before deploying. This is used by the sitemap, robots.txt, and OG image metadata.

SEO & Metadata

Metadata is managed via Next.js Metadata API in app/layout.tsx. Update title, description, keywords, and openGraph fields for your product.

  • sitemap.xml — auto-generated at /sitemap.xml. Add new routes in app/sitemap.ts.
  • robots.txt — auto-generated at /robots.txt.
  • OG image — generated at /opengraph-image via app/opengraph-image.tsx.
  • Favicon — generated at /icon via app/icon.tsx.
  • Apple touch icon — generated at /apple-icon via app/apple-icon.tsx.
